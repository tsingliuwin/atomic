# ⚛️ 原子理论 (Atomic Theory)

> **首个 AI Native 软件工程宣言：从“手工业”向“生物进化”的范式转型。**

## 🌟 愿景：代码即意图 (Code as Intent)
**原子理论**是为 AI Native 时代量身定制的革命性软件工程方法论。我们不再视软件为死板的逻辑堆砌，而是视其为一种**活的、可演化的生物系统**。在这种范式下，人类开发者从“手工艺人”转变为“意图的播种者”，而 AI 则负责功能“原子”的全生命周期管理。

## 🧬 核心哲学：代码生物学 (Code Biology)
我们从生物进化的视角重新定义软件开发：
- **原子即 DNA**：功能单元是系统的最小构建块。
- **Git 即基因测序**：每一次提交都是一次精准的基因突变。
- **自然选择**：自动化契约和测试决定了哪些代码能够“生存”并进入主干基因库。
- **语言自治**：AI 根据效率原则为每个任务选择最合适的“生物表达”（Rust, Python, Wasm 等），打破了单语言架构的垄断。

## 🚀 核心特性
- **原子溯源性**：每一个功能变更都映射到唯一的 Git Hash，实现“基因修复”（通过 Diff 分析自动修复 Bug）。
- **AMS (原子元数据规格书)**：一种机器可读的契约，允许 AI 在无需人工干预的情况下索引、理解和编排异构代码。
- **自主开发闭环**：AI 驱动的“意图拆解 -> 手术实现 -> 自检自证 -> 归档存证”。
- **语言无关编排**：通过 Wasm、Unix Sockets 或 FFI 无缝集成跨语言原子。

## 📁 项目结构
- **[`ATOMIC_THEORY.md`](./ATOMIC_THEORY.md)**：**宣言主文件**。所有法律、协议和哲学基础的终极信源。
- **[`ATOMIC_METADATA.json`](./ATOMIC_METADATA.json)**：**基因注册模板**。跟踪原子身份、类型和进化历史的标准 Schema。
- **[`ATOMIC_THEORY_DISCUSSION_LOG.md`](./ATOMIC_THEORY_DISCUSSION_LOG.md)**：**意图史记**。记录了理论演进背后的逻辑与辩论。

## 🛠 如何使用
1. **理解法律**：阅读 `ATOMIC_THEORY.md` 中的“DNA 定律”。
2. **初始化项目**：将 `ATOMIC_METADATA.json` 模板复制到您的子项目或 Demo 目录。
3. **赋能 AI**：将这些文档提供给您的 AI 代理（如 Gemini CLI），开启自主的原子化开发。
4. **遵守协议**：确保每次提交都遵循 `[Atom:<id>]` 语义格式，以维护基因序列。
